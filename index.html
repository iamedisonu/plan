6 Data & Prototype Limitations

The app has a real client-side state model, but every account, workspace, chat, message, preference, and lifecycle action is stored locally. Nothing is synchronized to a server.

What the browser stores

  • Local account details, password, and current session.
  • Per-user workspaces, spaces, chats, messages, and file metadata.
  • Theme, visual look, Archive state, and Bin timestamps.
  • Normalized migrations and recovery state after interrupted responses.

What is not connected

  • No server authentication, permissions service, or shared accounts.
  • No API, database, AI model, or live programme data.
  • No file-content upload, parsing, or document storage.
  • No live feed, SQL execution, validation job, or real citation retrieval.

Evidence behavior

The dedicated right-side rail displays SQL run, Rows, Documents, and Citations for the latest assistant message. Every value is placeholder context for demonstrating the intended layout.

File behavior

Only each selected file’s name, type, size, and modification timestamp are retained as references. File contents are never uploaded or read by this prototype.

Safety note

Do not enter a real or reused password, sensitive programme information, or confidential files. Clearing browser storage can remove all locally saved work.

8 Chat And Evidence

The current chat experience demonstrates lifecycle, organization, draft, and evidence interactions without making a network request.

Chat lifecycle

Create global or space chats, derive a title from the first message, rename manually, search, pin, archive, restore, move to Bin, restore from Bin, or delete permanently.

Draft protection

Untouched draft chats are reused, unsent composer content is protected during conflicting navigation, and workspace changes clear or preserve draft state deliberately.

Simulated answer

Circles generates a delayed local response after roughly 700 ms. The response can list selected file names, but it never reads those files and does not contact an AI model.

Evidence rail

The separate rail shows placeholder SQL status, rows, documents, and citations. During a reload, an unfinished answer becomes an interrupted message instead of remaining stuck.

10 Rules And Guardrails

These rules keep reviews accurate today and make future service integration safer.

Do

  • +Keep every placeholder metric, answer, source, and citation visibly labeled as demo content.
  • +Run npm test and npm run build before publishing.
  • +Preserve route fallbacks, keyboard behavior, draft guards, and recovery states.
  • +Keep the browser-only warning visible while local storage is used.
  • +Add future APIs behind a service layer with loading, empty, and error states.

Do Not

  • -Treat localStorage credentials or sessions as production authentication.
  • -Present sample Overview figures or Data Health states as live operations.
  • -Store real passwords, confidential records, or sensitive programme information.
  • -Claim that selected file contents are uploaded, parsed, or searched.
  • -Enable planned workspaces until their behavior and data contracts are implemented.
